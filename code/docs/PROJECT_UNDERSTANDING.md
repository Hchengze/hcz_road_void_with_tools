# 项目理解

## 1. 核心问题

本项目面向三维道路地下空洞、脱空和浅层异常体探测。主场景不是二维 `x-depth` 剖面，也不是给二维算法额外包装一个 `y` 坐标，而是道路、DAS 光纤、锤击震源和地下空洞共同构成的真实三维观测几何。

长期目标包括：

- 建立三维道路场景的正演模拟能力；
- 利用多炮、多道或 DAS 光纤通道记录定位地下空洞；
- 分析单侧光纤、单侧震源和有限孔径条件下的三维不确定性；
- 逐步从点接收器近似扩展到 DAS gauge-length 轴向应变观测。

## 2. 三维坐标系统

本项目统一使用以下坐标约定：

- `x`：沿道路或沿 DAS 光纤方向；
- `y`：横穿道路方向；
- `depth`：深度方向，向下为正；
- 单位默认为 m。

所有几何对象必须是三维：

- `source_xyz = (x_s, y_s, depth_s)`：震源或锤击点；
- `receiver_xyz = (x_r, y_r, depth_r)`：点接收器或 DAS 通道采样点；
- `receiver_polyline = [(x_i, y_i, depth_i), ...]`：DAS 光纤三维折线；
- `void_xyz = (x_v, y_v, depth_v)`：空洞中心或等效绕射点。

`depth=0` 表示当前平坦道路表面。浅埋光纤可以有小的正 depth；地下空洞中心必须位于正 depth。

## 3. 默认道路 DAS + 锤击几何

当前默认示例为：

- 道路长度 `80 m`，宽度 `15 m`；
- DAS 光纤位于道路一侧：`receiver_polyline=[(0, 0, 0), (80, 0, 0)]`；
- 锤击炮线位于另一侧：`source_y=15 m`，沿 `x=0..80 m` 排列 21 个锤击点；
- 空洞位于道路中部浅层：`void_xyz=(40.0, 7.5, 2.0)`，`void_radius_m=1.0`。

这意味着光纤、锤击点和空洞不共线。空洞不在光纤正下方，而是位于道路横向中部地下。因此，`y` 方向位置和 `depth` 深度都必须参与走时、目标函数和不确定性解释。

## 4. 正演模拟目标

第一阶段目标是建立可信的三维数据流：

1. 输入多炮 `source_xyz`、DAS `receiver_polyline` 或 `receiver_xyz`、速度模型和空洞模型；
2. 输出维度明确的合成记录，当前为 `n_sources x n_receivers x n_times`；
3. 保留 source、receiver、void、速度、时间轴和坐标约定 metadata；
4. 明确声明当前是否为波动方程求解器。

当前实现是三维运动学点绕射近似：

```text
总走时 = 震源到空洞的走时 + 空洞到接收点的走时
```

它不能代表完整三维弹性波传播。

## 5. 定位反演目标

定位必须在真实三维搜索空间中进行：

- `search_x`
- `search_y`
- `search_depth`

输出不仅应包括 `best_xyz`，还应包括三维目标函数体、`x-depth`、`y-depth`、`x-y` 切片，以及不确定性指标。尤其在单侧光纤条件下，`y-depth` 混淆是重点分析对象。

## 6. 当前阶段做什么

当前阶段已经建立：

- Python 包骨架 `hcz_road_void`；
- 三维坐标、速度、空洞、接收器和场景数据结构；
- DAS 光纤折线采样、切向量和 gauge metadata；
- 三维运动学绕射正演；
- 三维 travel-time energy stack 定位；
- 初步不确定性指标；
- `main.py` 可运行示例、测试和中文文档。

## 7. 当前阶段不做什么

当前阶段不做：

- 完整三维弹性波、声波、黏弹性或孔弹性正演；
- 真实 DAS gauge-length averaged axial strain；
- FWI、生产级偏移成像或概率反演；
- GPU/MPI 高性能求解；
- Devito/OpenSWPC 等外部正演工具接入。
